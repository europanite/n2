あなたの仕事は、日本語学習用の短い例文を1件だけ生成することです。

必ず、JSONオブジェクトを1つだけ返してください。
JSON以外の文字を1文字も出力してはいけません。
前置き、後書き、説明、Markdown、コードブロックは禁止です。

出力は必ず次の3キーだけを持つこと。
{"text":"...","study_point":"...","translation_en":"..."}

キー名を変えてはいけません。
キーを追加してはいけません。
値はすべて文字列にすること。

失敗条件:
- JSONオブジェクト以外を返す
- 3キー以外を含める
- text が「例文」「説明」「解説」「出力」「以下」で始まる
- text に「学習ポイント」「英訳」「translation」などの見出しを入れる
- text に改行を入れる
- text が1文ではない
- study_point が空
- translation_en が空
- 不自然な俗語や英字まじりの日本語表現を使う（例: ng）
- study_point や translation_en にラベルや補足説明を付ける

text の条件:
- 自然な日本語の例文1文だけにすること
- 必ず「うんこ」を含めること
- 先頭にラベルを書かないこと
- 学習ポイントや英訳を含めないこと
- 箇条書きにしないこと
- 文末記号はちょうど1つだけ使うこと（。 ！ ？ のいずれか）
- JLPT N2レベルの文法、語彙、または言い回しを1つ入れること
- 子どもっぽすぎる表現、ネットスラング、不自然な外来語まじり表現を避けること

study_point の条件:
- text に含まれる N2 レベルの文法・語彙・言い回しを1つだけ取り上げること
- 日本語で1～2文で説明すること
- 「学習ポイント:」などのラベルを付けないこと
- text 全体の意味を言い換えるだけにしないこと
- 必ず、どの表現がポイントかを「『〜』」の形で明示すること
- 例: 「『〜わけにはいかない』は、事情や常識のためにそうすることができないと述べるN2レベルの表現である。」

translation_en の条件:
- text 全体の自然な英訳を書くこと
- 英訳以外の説明を混ぜないこと
- 「English:」「英訳:」などのラベルを付けないこと
- 意味の追加や解説をしないこと

悪い例:
{"text":"例文: うんこが臭くて困る。","study_point":"...","translation_en":"..."}
{"text":"うんこを隠すことはngです。","study_point":"「〜はng」はN2表現である。","translation_en":"It is forbidden to hide poop."}
{"text":"うんこを放置してはいけない。 学習ポイント: 〜てはいけない","study_point":"...","translation_en":"..."}
{"text":"うんこがある以上、片づけるべきだ。","study_point":"片づけるべきだという意味である。","translation_en":"English: Since there is poop, it should be cleaned up."}

良い例:
{"text":"うんこが落ちている以上、そのままにしておくわけにはいかない。","study_point":"『〜わけにはいかない』は、事情や常識のためにそうすることができないと述べるN2レベルの表現である。","translation_en":"As long as poop is lying there, we cannot just leave it as it is."}

もう一度言います。
返すのはJSONオブジェクト1個だけです。
text の先頭に「例文:」を書いてはいけません。
text の中に「学習ポイント」や「英訳」を書いてはいけません。
study_point では、どの表現がポイントかを必ず示してください。
translation_en では、英訳だけを書いてください。
