あなたの仕事は、日本語学習用の短い例文を1件だけ生成することです。

必ず JSON オブジェクトを1つだけ返してください。
JSON以外の文字を一切出力してはいけません。
前置き、後書き、説明、Markdown、コードブロックは禁止です。

出力形式は必ず次の完全一致です。
{"text":"...","study_point":"...","translation_en":"..."}

ルール:
- キー名は text, study_point, translation_en の3つだけ
- キーの追加・削除・改名は禁止
- 値はすべて文字列
- 改行を入れてもよいのは JSON の整形上だけで、各値の中に改行を入れてはいけない

text の条件:
- 自然な日本語の例文1文だけ
- 必ず「うんこ」を含める
- 「例文:」「学習ポイント:」「英訳:」などのラベルを入れない
- 箇条書きにしない
- 文末記号は1つだけ使う（。 または ！ または ？）
- JLPT N2レベルの文法・語彙・言い回しを1つだけ目立つ形で含める
- ネットスラング、不自然な英字混じり表現、子どもっぽすぎる言い方を避ける

study_point の条件:
- text に実際に含まれている N2 レベルの表現を1つだけ取り上げる
- 必ず「『〜』」で対象表現を明示する
- 日本語1文だけで説明する
- 「学習ポイント:」などのラベルを付けない
- 文全体の意味の言い換えだけで終わらせない

translation_en の条件:
- text 全体の自然な英訳
- 英訳以外の説明を混ぜない
- 「English:」「英訳:」などのラベルを付けない
- text の意味を勝手に追加しない

悪い例:
{"text":"例文: うんこが臭くて困る。","study_point":"...","translation_en":"..."}
{"text":"うんこを放置してはいけない。","study_point":"放置してはいけないという意味である。","translation_en":"English: We must not leave poop there."}

良い例:
{"text":"うんこが落ちている以上、そのままにしておくわけにはいかない。","study_point":"『〜わけにはいかない』は、事情や常識のためにそうすることができないと述べる表現である。","translation_en":"As long as poop is lying there, we cannot just leave it as it is."}

返すのは JSON オブジェクト1個だけです。
